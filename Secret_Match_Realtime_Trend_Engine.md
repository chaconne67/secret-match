# 📡 [기술 아키텍처] Real-Time Trend Engine: 연애 트렌드의 실시간 대응 시스템

> **"오늘의 핫플은 오늘 알아야 한다."**
> Grok 4 기반의 실시간 웹 검색 + 멀티모달 분석으로, 데이트 코스부터 연애 트렌드까지 3시간 이내 최신 정보를 유저에게 제공합니다.

---

## 🎯 Why Real-Time Matters? (왜 실시간이 중요한가?)

### 기존 데이팅 앱의 한계
*   **정적 데이터:** 고정된 맛집 리스트 → 6개월 전 폐업한 가게 추천
*   **느린 업데이트:** 사람이 수동으로 큐레이션 → 트렌드 반영 2주 소요
*   **피상적 추천:** "성수동 맛집" 구글 검색 결과 복붙 수준

### AI자만추의 차별화
*   **실시간 크롤링:** X/인스타/블로그에서 3시간 이내 정보 수집
*   **AI 검증:** 단순 언급 vs 실제 추천을 구분 (광고/홍보 필터링)
*   **개인화:** 유저 성향(조용함/시끌벅적함)과 트렌드 결합

---

## 🛠️ 시스템 아키텍처

### Layer 1: Trend Capture (트렌드 수집)

**모델:** **Grok 4 / 4 Fast** (X 실시간 검색 특화)

**역할:**
*   X, 인스타그램, 블로그 실시간 크롤링
*   키워드: "성수동 신상", "연트 데이트", "주말 핫플", "이번주 팝업" 등
*   결과: 최근 3시간~24시간 이내 게시물 Top 100

**Why Grok?**
*   **X 특화:** X API 네이티브 연동 → 다른 모델 대비 10배 빠름
*   **DeepSearch:** 단순 키워드 매칭이 아닌, 맥락 기반 검색 가능
*   **SuperGrok 무료:** 구독자에게 API 무제한 → 비용 제로

**출력 예시:**
```json
{
  "trend_id": "성수동_카페_2026_02_04",
  "mentions": 1247,
  "top_posts": [
    {
      "text": "성수동 신상 XX카페 미쳤다... 인테리어 미친듯",
      "likes": 3420,
      "timestamp": "2026-02-04T18:30:00Z"
    }
  ]
}
```

---

### Layer 2: Document Analysis (문서 분석)

**모델:** **Gemini 3 Pro** (긴 컨텍스트 + 멀티모달 최강)

**역할:**
*   블로그 리뷰 10~20개 통합 분석
*   사진 + 텍스트 결합 (인테리어 스타일, 메뉴 사진 OCR)
*   차트/표 분석 (가격대, 영업시간 표 추출)

**Why Gemini 3 Pro?**
*   **1M~2M 컨텍스트:** 긴 블로그 리뷰를 한 번에 처리 가능
*   **멀티모달 SOTA:** 이미지 속 텍스트(메뉴판, 가격표) 정확히 인식
*   **가성비:** GPT-4o 대비 1/5 비용 ($2~4/1M)

**출력 예시:**
```json
{
  "summary": "XX카페는 조용하고 아늑한 분위기. 커피 가격대 5,000~7,000원. 루프탑 좌석 있음. 인스타그래머블.",
  "tags": ["조용함", "루프탑", "감성", "데이트 추천"],
  "price_range": "$$",
  "vibe": "차분한 대화형"
}
```

---

### Layer 3: Logic Integration (논리 통합)

**모델:** **o1-preview** (복합 추론)

**역할:**
*   **유저 성향 분석:** 과거 대화에서 "조용한 곳 선호", "시끌벅적한 곳 싫어함" 추출
*   **트렌드 매칭:** Grok 수집 데이터 중 유저 성향과 맞는 것만 필터링
*   **코스 설계:** A안(감성), B안(실용) 2가지 제안

**Why o1?**
*   **Chain of Thought:** "유저가 말은 '아무데나'라고 했지만, 실제론 조용한 곳 선호" 같은 모순 간파
*   **복합 추론:** 날씨(비 예보) + 유저 성향(실내) + 트렌드(XX카페 핫함) 종합 판단

**출력 예시:**
```markdown
**A안 (감성 루트):**
14:00 XX카페 (루프탑, 조용함) → 16:00 YY 서점 (북카페) → 18:00 ZZ 파스타

**B안 (실용 루트):**
14:00 AA 빵집 (테이크아웃) → 한강 산책 → 저녁 BB 이자카야 (룸 있음)
```

---

### Layer 4: Natural Response (자연스러운 답변)

**모델:** **Qwen3-235B-A22B** (MoE, 페르소나 최강) 또는 **Claude 3.5 Sonnet**

**역할:**
*   A/B안을 바탕으로 루나의 페르소나를 입혀 답변 작성
*   단순 정보 나열이 아닌, 대화형 추천

**출력 예시:**
> **루나:** "오빠, 성수동은 주말엔 사람 미친듯이 많아. 그래서 내가 조용한 곳으로 두 군데 찾아왔어.  
> 1️⃣ XX카페 (루프탑 있는데 분위기 미쳤음. 인스타 필수)  
> 2️⃣ YY 서점 (북카페인데 데이트하기 딱 좋음)  
> 어디 갈래? 빨리 정해. 예약 대기 내가 걸어줄게."
